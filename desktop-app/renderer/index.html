Import z historii klienta League

Pozwala doimportować mecze, w których apka nie była uruchomiona w chwili zakończenia gry (np. gdy Cię przy tym nie było) - z pełnymi statystykami, tak jak przy zwykłym automatycznym przechwytywaniu. Wymaga uruchomionego i zalogowanego klienta League of Legends. To lokalna historia zapisana przez sam klient - dla starszych dat lista może się urwać wcześniej, niż faktyczna liczba rozegranych meczów by sugerowała, jeśli klient nie trzyma ich już w swoim cache.

Skanowanie zakresu Game ID

Ostatnia deska ratunku, gdy powyższa historia klienta się urywa (twardy limit lokalnego cache League): sprawdza po kolei każdy Game ID w podanym zakresie przez /lol-match-history/v1/games/{id} i zapamiętuje te, w których brało udział aktualnie zalogowane konto. Ważne ograniczenie: to konto musi było osobiście grać w danym meczu - meczów hostowanych/nagranych przez kogoś innego, w których to konto nie grało, ten skan nigdy nie znajdzie (do tego służy import .rofl / starych plików JSON wyżej).

gameId jest globalnym licznikiem dla całego serwera, nie jest powiązany z datą ani z Twoim kontem - zakres odpowiadający kilku dniom grania może obejmować miliony cudzych meczów do przelecenia po kolei. Dlatego skan działa powoli (ograniczona liczba zapytań na sekundę, żeby nie obciążać serwerów Riota) i może trwać godzinami albo dniami - da się go bezpiecznie zatrzymać i wznowić później od ostatniego sprawdzonego ID, nawet po zamknięciu apki.

Brak zapisanego postępu skanowania.

Import z plików .rofl

Import próbuje najpierw historii klienta League po identyfikatorze gry z nazwy pliku (pełniejszy kształt danych - osobne bany drużynowe), a jeśli to się nie uda (np. zalogowane konto nie brało udziału w tej grze), sięga po statystyki graczy zaszyte wprost w samym pliku .rofl. Kolumna "Źródło danych" w tabeli meczów pokazuje, której metody ostatecznie użyto. Przed zapisem zobaczysz podgląd danych meczu i będziesz mógł zdecydować, czy go zaimportować.
